Featured project

Outside of Roblox, I build this.

Amadeus
local-first AI companion · Python / FastAPI
v0.13.0 · finished, for now

A fully local AI companion inspired by the Amadeus system from Steins;Gate 0. Voice conversation, long-term memory, a live Live2D avatar, running entirely on my own hardware with no cloud dependency.

The chatbot itself wasn't the hard part. The memory system is what took the longest: seven different memory types (working, episodic, semantic, relationship, and a few more) scored through hybrid retrieval that weighs recency, importance, and similarity together. On top of that there's a full voice pipeline with real-time barge-in, and an avatar layer that has to work around the rendering framework to keep control of its own parameters.

FastAPI + WebSocketsSQLite / FTS5 Local LLM (Ollama)faster-whisper Kokoro TTSSilero VAD Live2DMemory Consolidation
View source on GitHub

Non-commercial fan project. I built it to find out whether a full companion AI stack could run on ordinary consumer hardware. It can, with the right tradeoffs.

Voice pipeline

16kHz browser audio to VAD to STT to local LLM to TTS, with barge-in cancellation so you can actually interrupt it mid-sentence.

Memory system

Post-session LLM consolidation handles deduping, catches contradictions, and keeps emotional shifts bounded so it doesn't spiral after a bad conversation.

Live2D avatar

A parameter injection hook overrides the framework's own update cycle, driven by inline expression tags that get stripped out before the LLM's reply is spoken.

Built to not fall over

Startup recovery, job stampede limits, a warmup-based GPU fallback. Unglamorous work, but it's the reason the thing stays up.
